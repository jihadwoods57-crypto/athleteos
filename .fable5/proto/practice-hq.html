Practice HQ

The bare Trainer Profile settings page becomes a real business command center. It shows the trainer's true server identity (survives a reinstall, never another persona), turns the client code into a one-tap invite loop with QR + native share, and fixes the back button that dropped trainers into a random athlete profile. Tap the states below.

Fix 1 Back button no longer lands in a stranger's profile

The coach, trainer and parent dashboards are tab roots that passed 'profile' to the back chevron, sending the user into the athlete profile. Tab roots get no back chevron; sub-screens like Practice HQ point back to the trainer dashboard.

Fix 2 Real, persistent trainer identity

Identity and code hydrate from the server on sign-in, full_name from profiles plus practice id / name / client-code, into RT. On a fresh device the profile shows the real business, never the hardcoded "Tracy Boone" fallback, and never "No code yet" when a code exists.

New The invitation loop

Real client code, a scannable QR of the join link, Copy, and native Share, built on the existing SHARE bridge and clipboard. Onboarding a client is one tap. Production inlines a dependency-free QR generator; this prototype renders the exact layout with a seeded matrix.

Roadmap The rest of Practice HQ
