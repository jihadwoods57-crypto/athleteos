## 1. What exists today (read first, per the ticket)

The current screen (`notifications.js`) is already honest and grace-calibrated: it renders the exec-engine feed
(overdue ‚Üí next-up ‚Üí assigned ‚Üí wins ‚Üí celebration) with a tiered streak-at-risk row unshifted to the top
(amber/flame when grace is spent, blue/shield when intact), plus humanized level pills and a caught-up empty state.

But it is a **flat list, not an experience**. Concretely, three gaps:

1. **No read/unread state per row.** Every row looks identical forever. Read state is a single boolean
   (`RT.notifsRead`, state.js:87/281): the bell badge zeroes the instant you open the screen, but nothing on any
   row changes, and re-opening shows the same rows with no "you've seen this" signal.
2. **Every timestamp is the literal string `'now'`** (state.js ‚Äî every `when:'now'`). The time column is decorative,
   so there is nothing to group or scan by.
3. **The two-section scaffold is vestigial.** `earlier` is hard-coded `[]` (state.js:1406), so the "New" eyebrow
   labels the only group that ever exists. `state-demo` empty state is a dashed placeholder box.

The bell + badge entry point (`appHead`, components.js:158/165) is the one part already doing real work: the badge
count is `S.unreadNotifs` = `notifications.new.length` until read. That number is honest. Keep it; refine it.

## 2. The one binding decision: full screen, not a drawer

At 390px I keep this a **routed full screen**, and reject a panel/drawer. Reasons:

- These notifications are **actionable accountability moments** ‚Äî every row routes you *into* the app (log a meal,
  open a requirement, view the score breakdown). Drawers earn their keep for low-commitment glanceable peeks; a
  surface you act on and navigate away from wants the whole viewport and room per row.
- It is **already** a hash route (`#notifications`, `tab:'home'`) that participates in back-nav and deep-linking.
  A bottom-sheet fork would add a scrim + dismiss gesture for zero gain and break the app's single navigation model.
- **Consistency (quality bar #1):** home / plan / progress / profile are all full screens. A notifications drawer
  is a per-surface fork ‚Äî exactly what SETTLED memory forbids ("no per-screen style forks").

## 3. The upgrade (all inside the existing dark language + tokens)

Every value below is a real token from `css/tokens.css` ‚Äî no new colors, radii, or type. Reuses `.notif`, `.nic`,
`.level-tag`, `.eyebrow`, `.sidebox` structure.

**Per-row read / unread states** (the core of the ticket):
- **Unread:** full 1px border in the row's level color (`--amber/blue/green/red-border`) + a faint top-inset wash
  of the same hue at ~5% + a 9px level-colored **unread dot** in the meta column, title at full `--text` / 800.
  *Deliberately not a side-stripe* (banned): the accent is a full border + a dot, never a left rail.
- **Read:** flat `--surface-1`, `--hairline` border, icon tile at 0.5 opacity, title dimmed to `--text-2`
  (still AA, 8.3:1), body to `--text-3` (5.7:1, AA), dot replaced by the route chevron.
- Read state is **never conveyed by color alone**: weight, the dot‚Üíchevron swap, and the group all reinforce it.

**Live header control cluster** (new, honest today): an unread-count summary (`"6 unread"`) with a **Mark all read**
text action (reuses the `.eyebrow .link` blue). This maps 1:1 to the *existing* `readNotifs()` ‚Äî zero new data. When
0 unread it becomes a green `‚úì All caught up` line and the button disables.

**Real recency groups** that collapse: two variants in the prototype (see ¬ß4). Groups only render when they have
rows, so with today's all-`'now'` data the screen still reads correctly (one group) ‚Äî the grouping *upgrades*
automatically the moment real timestamps exist (see openQuestions).

**Swipe-to-dismiss** per row (drag left ‚Üí red "Dismiss" backing ‚Üí undo toast). Designed as the target; needs
persistence plumbing to survive reload (openQuestions).

**Upgraded empty state:** a green check-ring with a soft radial wash, `"You're all caught up"`, one honest line, and
a 3-item preview of what lands here (meal/weigh-in nudges, coach requirements, streak reminders). Replaces the dashed
placeholder box.

**Bell badge refinement:** already real; cap the count at `9+`, hold a 44px tap target (currently 42px ‚Äî see ¬ß6),
carry a proper `aria-label` (`"Notifications, 6 unread"`), and keep it clearing to nothing at 0 so a lit bell always
means something genuinely needs the athlete.

## 4. Two variants (toggle in the prototype) ‚Äî recommendation: A

- **A ¬∑ By recency (RECOMMENDED).** Groups: *New / Earlier today / This week*. A notifications feed is a log, and a
  log is chronological. It matches the calm "accountability moments, not spam" ethos and keeps the taxonomy distinct
  from Home.
- **B ¬∑ By consequence.** Groups: *Needs you / Streak on the line / Wins*. More opinionated and premium, but it
  **duplicates the taxonomy Home and the Action Hub already own** (urgency ordering) and blurs the two surfaces.
  Presentation-only reordering, so it's safe ‚Äî but it earns its keep less.

Recommend **A**. It is the lower-risk, more honest read of what this surface *is*, and it's the one that gets better
for free once real timestamps land.

## 6. Accessibility / mobile-first (held, per quality bar)

- Verified at **390√ó844** in a real browser across all 5 states √ó 2 variants + the bell entry view.
- **Tap targets:** rows tall (>44px); Mark all read, back, settings all ‚â•44px. Bell + back bumped **42‚Üí44px** ‚Äî
  a floor-*hold* recommendation, not a regression.
- **Contrast (AA):** read-state title `--text-2` (8.3:1), read body `--text-3` (5.7:1); never dimmer than `--text-3`.
  Level-colored dots pair with weight + grouping so state is never color-only.
- `prefers-reduced-motion` respected (all transitions/animations disabled). Rows are `role="button"`, keyboard
  operable (Enter/Space), visible focus rings.
