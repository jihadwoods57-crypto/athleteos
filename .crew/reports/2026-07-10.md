### ✅ `crew/2026-07-10-i1` — Stop fabricating the parent Weekly-Compliance headline from seeded demo data
- **Pillar:** reliability / honesty · **Severity:** high · **Constitution:** *"demo data is never the
  default state" + "engagement is earned, never manufactured."*
- **Defect:** `trendSeries()` left-pads a sparse real history with the hardcoded
  `SEEDED_LEAD = [82,80,83,86,88,85]` (every value ≥ the 80 compliance bar). `weeklyCompliance()` and
  `nutritionTrend()` then computed their **headline numbers from that padded series**, so a *real*
  parent whose athlete has < 6 days of history saw e.g. **"5 of 6 days on plan · 84%"** and a row of
  green ✓ dots — entirely manufactured from demo padding. The behavior was **untested** (the only
  `weeklyCompliance([], …)` test asserted the today-marker, never the fabricated aggregate).
- **Fix:** aggregates now count **real completed days only** (leading seed excluded via
  `realTrendDays`). Seeded points are flagged (`ComplianceDay.seeded`, `NutritionTrend.seededBefore`)
  and `ParentView` renders them as neutral placeholders instead of green checks / filled bars; a
  no-history athlete shows an honest **"Building this week — no completed days yet"** and `—` instead
  of a made-up percent. Padded series kept for chart shape so dots still match the trend line.
- **Tests:** corrected 2 tests that encoded the fabrication; added 4 honesty tests. `npm run verify`
  green (tsc + jest + iOS bundle). Files: `src/core/history.ts`, `src/core/history.test.ts`,
  `src/screens/roles/ParentView.tsx`.
- **Revert:** `git reset --hard crew/2026-07-09-i1`.
